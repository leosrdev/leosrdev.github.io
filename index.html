Step-by-Step Example: Finding Private Key d (RSA Context)

Using the Extended Euclidean Algorithm to find the private exponent d in RSA

Problem Setup

Given RSA parameters:

$$p = 5, \quad q = 11$$

$$e = 3 \text{ (public exponent)}$$

Calculate:

$$n = p \times q = 5 \times 11 = 55$$

$$\varphi(n) = (p-1)(q-1) = 4 \times 10 = 40$$

Find:

$$d \text{ such that } e \cdot d \equiv 1 \pmod{\varphi(n)}$$

$$3 \cdot d \equiv 1 \pmod{40}$$

Variable Meanings:
• $$p, q$$ = two distinct prime numbers
• $$n = pq$$ = RSA modulus
• $$\varphi(n)$$ = Euler's totient function
• $$e$$ = public exponent (must be coprime with $$\varphi(n)$$)
• $$d$$ = private exponent (multiplicative inverse of $$e$$)

Step 1: Forward Pass (Euclidean Algorithm)

$$40 = 3 \times 13 + 1$$
$$q_1 = 13, \quad r_1 = 1$$
$$\gcd(40, 3) = \gcd(3, 1)$$
$$3 = 1 \times 3 + 0$$
Remainder is $$0$$, so $$\gcd(e, \varphi(n)) = 1$$ ✓
(Inverse exists because gcd = 1)

Step 2: Backward Substitution

$$1 = 40 - 3 \times 13$$
From Step 1, rearrange: $$1 = 40 - (3 \times 13)$$
This means: $$1 = 1 \times 40 + (-13) \times 3$$

Step 3: Reduce Modulo

$$(-13) \times 3 \equiv 1 \pmod{40}$$
From backward substitution
$$d \equiv -13 \equiv 27 \pmod{40}$$
Since $$-13 + 40 = 27$$, we get positive: $$d = 27$$

Verification

$$e \times d = 3 \times 27 = 81 = 2 \times 40 + 1 \equiv 1 \pmod{40}$$ ✓
Success! $$d = 27$$ is the correct private exponent

Big O Complexity Reference

Common Scenario Complexity Explanation
Single simple operation $$O(1)$$ Constant time: accessing array element, hash lookup
Simple loop through array $$O(n)$$ Linear: iterates each element once
Nested loops (dependent) $$O(n^2)$$ Quadratic: nested loops multiply the complexity
Triple nested loops $$O(n^3)$$ Cubic: three levels of nesting
Binary search $$O(\log n)$$ Logarithmic: divides problem in half each iteration
Merge Sort $$O(n \log n)$$ Linearithmic: log n levels, each doing n work
Quick Sort (average) $$O(n \log n)$$ Linearithmic: divide and conquer approach
Bubble Sort $$O(n^2)$$ Quadratic: compares adjacent elements repeatedly
Recursive fibonacci $$O(2^n)$$ Exponential: each call branches into 2 calls
Generating permutations $$O(n!)$$ Factorial: explores all possible arrangements
Prime check (trial division) $$O(\sqrt{n})$$ Square root: only check divisors up to √n
Matrix multiplication (naive) $$O(n^3)$$ Cubic: three nested loops over matrix dimensions
