Big O Notation Analysis

Learn to analyze 12 different algorithmic patterns

Example 1
function getFirstElement(array) {
    return array[0]
}
Constant Time
$$O(1)$$

How to Analyze

  1. Identify loops - No loops present
  2. Count operations - 1 access operation (array[0])
  3. Check input dependency - Execution time doesn't depend on input size
  4. Conclusion - Same time regardless of array size = $$O(1)$$
Example 2
function findMax(array) {
    max = array[0]
    for i = 1 to array.length {
        if array[i] > max {
            max = array[i]
        }
    }
    return max
}
Linear Time
$$O(n)$$

How to Analyze

  1. Identify loops - One for loop
  2. Count iterations - Loop runs n times (array.length)
  3. Check nested operations - Inside loop: constant operations (comparison, assignment)
  4. Calculate total - $$n \times 1 = n$$ operations
  5. Conclusion - Time directly proportional to input size = $$O(n)$$
Example 3
function bubbleSort(array) {
    for i = 0 to array.length {
        for j = 0 to array.length - 1 {
            if array[j] > array[j+1] {
                swap(array[j], array[j+1])
            }
        }
    }
}
Quadratic Time
$$O(n^2)$$

How to Analyze

  1. Identify loops - Two nested for loops
  2. Count outer loop - Runs n times
  3. Count inner loop - Also runs n times for each outer iteration
  4. Calculate total - $$n \times n = n^2$$ operations
  5. Conclusion - Nested loops multiply = $$O(n^2)$$
Example 4
function binarySearch(sortedArray, target) {
    left = 0
    right = sortedArray.length - 1
    while left <= right {
        mid = (left + right) / 2
        if sortedArray[mid] == target {
            return mid
        } else if sortedArray[mid] < target {
            left = mid + 1
        } else {
            right = mid - 1
        }
    }
}
Logarithmic Time
$$O(\log n)$$

How to Analyze

  1. Identify loops - One while loop
  2. Check what happens - Search space is halved each iteration
  3. Count iterations - To reduce n to 1, divide by 2 repeatedly: $$\log_2(n)$$ times
  4. Conclusion - Search space halves each time = $$O(\log n)$$
Example 5
function mergeSort(array) {
    if array.length <= 1 {
        return array
    }
    mid = array.length / 2
    left = mergeSort(array[0 to mid])
    right = mergeSort(array[mid+1 to end])
    return merge(left, right)
}
Linearithmic Time
$$O(n \log n)$$

How to Analyze

  1. Identify recursion - Divides problem in half each level
  2. Count levels - $$\log_2(n)$$ levels of recursion
  3. Work per level - Merge operation processes all n elements at each level
  4. Calculate total - $$n \text{ elements} \times \log(n) \text{ levels} = n \log n$$ operations
  5. Conclusion - Divide-and-conquer with linear merge = $$O(n \log n)$$

Cubic Time
$$O(n^3)$$

How to Analyze

  1. Identify loops - Three nested for loops
  2. Count each loop - Outer loop: n, Middle loop: n, Inner loop: n
  3. Multiply iterations - $$n \times n \times n = n^3$$
  4. Conclusion - Triple nested loops multiply = $$O(n^3)$$
Example 7
function fibonacci(n) {
    if n <= 1 {
        return n
    }
    return fibonacci(n-1) + fibonacci(n-2)
}
Exponential Time
$$O(2^n)$$

How to Analyze

  1. Identify recursion - Makes two recursive calls
  2. Trace calls - Each call branches into 2 more calls
  3. Depth - Recursion depth is n
  4. Calculate - $$2 \text{ branches per level} \times n \text{ levels} = 2^n$$ calls
  5. Conclusion - Each call branches exponentially = $$O(2^n)$$
Example 8
function generatePermutations(array, start = 0) {
    if start == array.length - 1 {
        print(array)
        return
    }
    for i = start to array.length {
        swap(array[start], array[i])
        generatePermutations(array, start + 1)
        swap(array[start], array[i])
    }
}
Factorial Time
$$O(n!)$$

How to Analyze

  1. Identify task - Generate all permutations
  2. Count possibilities - Total permutations of n elements = $$n!$$
  3. Recursion structure - Each position has decreasing choices
  4. Calculate - $$n \times (n-1) \times (n-2) \times \ldots \times 1 = n!$$
  5. Conclusion - Must explore all combinations = $$O(n!)$$
Example 9
function hasDuplicate(array) {
    for i = 0 to array.length {
        for j = i + 1 to array.length {
            if array[i] == array[j] {
                return true
            }
        }
    }
    return false
}
Quadratic Time
$$O(n^2)$$

How to Analyze

  1. Identify loops - Two nested for loops
  2. Outer loop - Runs n times
  3. Inner loop - Runs (n-i) times on average, still $$O(n)$$
  4. Multiply - $$n \times n = n^2$$ in worst case
  5. Conclusion - Compare elements pairwise = $$O(n^2)$$
Example 10
function isSquareNumber(n) {
    for i = 0 to sqrt(n) {
        if i * i == n {
            return true
        }
    }
    return false
}
Square Root Time
$$O(\sqrt{n})$$

How to Analyze

  1. Identify loop - One for loop
  2. Check limit - Loop goes from 0 to $$\sqrt{n}$$
  3. Count iterations - Loop runs $$\sqrt{n}$$ times total
  4. Conclusion - Loop bound is square root of input = $$O(\sqrt{n})$$
Example 11
function append(dynamicArray, element) {
    if dynamicArray.size == dynamicArray.capacity {
        // Resize: copy all elements to new larger array
        newArray = new Array[capacity * 2]
        for i = 0 to size {
            newArray[i] = array[i]
        }
        array = newArray
    }
    array[size] = element
    size++
}
Amortized Constant
$$O(1) \text{ Amortized}$$

How to Analyze

  1. Identify condition - Resize happens rarely (every doubling)
  2. Analyze single call - Most calls: $$O(1)$$, but occasional resize: $$O(n)$$
  3. Amortize - Spread resize cost across n append operations: $$n \text{ operations} / n \text{ appends} = O(1)$$
  4. Conclusion - Average cost per operation is constant = $$O(1) \text{ Amortized}$$
Example 12
function linearSearch(array, target) {
    for i = 0 to array.length {
        if array[i] == target {
            return i
        }
    }
    return -1
}
Linear Time
$$O(n)$$

How to Analyze

  1. Identify loop - One for loop checking each element
  2. Best case - Element at first position: $$O(1)$$
  3. Worst case - Element at end or not found: $$O(n)$$
  4. Average case - Element somewhere in middle: $$O(n/2) = O(n)$$
  5. Conclusion - Must potentially check all elements = $$O(n)$$
